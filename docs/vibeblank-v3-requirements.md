# 黑码码 V3 需求文档

## 1. 版本目标

V3 把黑码码从「菜单栏 + 固定快捷键」升级为一套可配置触发系统，让用户短暂离开时能更自然地开启黑屏模式，同时保持底层 AI agent、构建、终端和本地服务继续运行。

V3 的主张文案为「让 AI 悄悄帮你干活」。功能名仍使用「黑屏模式」，避免用户误解为系统锁屏、身份验证或官方屏幕保护程序。

## 2. 官方触发角 Spike 结论

需求优先考虑接入 macOS 官方触发角。当前实现结论：

- 不使用私有 API。
- 不直接改写 Dock 或 System Settings 的私有配置。
- 本机 SDK 可提供 ScreenSaver 框架用于实现屏保 bundle，但这会变成另一条分发和运行路径，不能直接复用黑码码现有菜单栏遮罩、设置、Esc 兜底和多显示器策略。
- 未发现稳定公开接口可让第三方 app 把自定义动作注册到 macOS 官方 Hot Corners 下拉菜单。

因此 V3 落地内置触发角 fallback：黑码码运行时自行检测鼠标进入指定屏幕角落，并触发黑屏模式。

## 3. 功能需求

### 3.1 登录时启动

- 默认开启「登录时自动启动黑码码」。
- 新用户和 V2 升级用户都默认开启。
- 用户可在设置页关闭。
- 使用 Apple `SMAppService.mainApp` 管理登录项。
- 如果系统要求用户批准或本地签名不满足要求，设置页和菜单栏显示状态，但不影响菜单栏、Esc 和黑屏功能。

### 3.2 Command 三连

- 默认启用。
- 默认识别任意 Command 三连。
- 设置中可选择任意 Command、左 Command 或右 Command。
- 约 0.8 秒内连续按下三次才触发。
- 触发后执行切换：未黑屏则开启，已黑屏则退出。
- 如果系统需要辅助功能权限，设置页显示引导，菜单栏提示权限状态。

### 3.3 自定义组合键

- 默认关闭。
- 默认候选为 `Control + Option + Command + B`。
- 用户可在设置页录制新的组合键。
- 保存前尝试注册组合键；注册失败则提示冲突并不保存。
- 不维护系统快捷键黑名单，不禁止用户选择可注册的组合键。
- 成功启用后执行切换：未黑屏则开启，已黑屏则退出。

### 3.4 内置触发角

- 默认关闭。
- 用户可选择左上角、右上角、左下角、右下角。
- 任意显示器进入所选角落都可触发。
- 触发角只开启黑屏，不退出黑屏。
- 指针需要离开角落后才可再次触发。
- 每次触发后至少 2 秒冷却，避免重复触发。

## 4. 非目标

- 不替代 macOS 锁屏、Touch ID 或密码验证。
- 不提供官方屏幕保护程序 bundle 分发。
- 不使用私有 API 接管系统官方 Hot Corners。
- 不做系统快捷键黑名单维护。
- 不新增账号、团队管理、通知集成或 agent 状态监听。

## 5. 验收标准

1. 首次启动和 V2 升级后，登录时启动默认开启，设置中可关闭。
2. 默认只有 Command 三连键盘触发启用，旧组合键默认关闭。
3. 任意 Command 三连可切换黑屏；切换为左/右 Command 后只响应对应按键。
4. 已占用组合键保存失败并显示冲突；成功组合键可开启或退出黑屏。
5. 触发角默认关闭；开启后进入所选角只开启黑屏，黑屏中再次进入角落不退出。
6. 权限不足时 Command 三连显示引导状态，不影响菜单栏、Esc 和已可用组合键。
7. `swift run VibeBlankCoreChecks` 和 `swift build --product VibeBlank` 通过。
8. `bash scripts/package_app.sh` 可生成 `.app`、`.zip` 和 `.dmg`。
