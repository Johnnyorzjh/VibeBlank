# VibeBlank V2 需求文档

## 1. 版本目标

V2 在 V1 的“快速遮住屏幕、不中断本机任务运行”基础上，补充两类体验优化：

1. 黑屏开启和退出时增加有方向感的渐变转场，让从亮到黑、从黑到亮的过程更有情绪张力。
2. 将应用内用户可见界面改为中文，降低理解成本，让菜单、设置页、遮罩状态文案保持一致。
3. 中文品牌名定为「黑码码」，并提供一个简洁的小黑马图标，强化产品记忆点。

V2 不改变 VibeBlank 的产品边界：它仍然是视觉隐私辅助工具，不是系统锁屏、认证工具或显示器电源管理工具。

## 2. 当前项目背景

V1 已实现 macOS 菜单栏应用，主要模块如下：

- `OverlayManager` 负责选择目标屏幕、创建和关闭遮罩窗口。
- `OverlayWindow` 负责全屏窗口、事件拦截、Esc 兜底退出。
- `OverlayContentView` 负责黑色遮罩和时间/状态/自定义文字展示。
- `AppDelegate` 负责菜单栏入口、菜单文案、快捷键状态和设置窗口。
- `SettingsView` 与 `AppSettings` 负责设置页与枚举展示名称。

V2 动画需求主要影响遮罩展示层，中文界面需求主要影响菜单栏、设置页、默认状态文案和窗口标题。

## 3. 需求理解

### 3.1 黑屏和变亮转场

用户希望黑屏和恢复变亮不再是硬切，而是有“渐变动作”。需求中的“从四周往中间突然亮”可以明确理解为退出黑屏时，屏幕亮起从四周边缘向中心推进。

“变黑”方向存在轻微语义歧义，V2 建议采用与退出相呼应的入场设计：开启黑屏时，黑色从屏幕四周边缘向中心快速包围，中间最后变黑。这样能制造被遮罩快速合拢的情绪，同时仍然符合隐私遮挡的目标。

### 3.2 中文界面

当前项目内大多数面向用户的字符串为英文。V2 需要将应用运行时用户能看到的界面统一改为中文，中文品牌名使用「黑码码」。

`VibeBlank` 作为 Swift package、可执行文件、内部技术名和必要兼容名保留，不作为主要用户界面标题展示。

### 3.3 小黑马图标

「黑码码」需要一个简洁的小黑马图标。图标应服务于 macOS 菜单栏和 app bundle，优先保证小尺寸辨识度，而不是复杂插画表现。

建议方向：简化黑马侧头像或奔跑轮廓，马鬃和轮廓带轻微代码感切面。图形以黑色为主，作为菜单栏图标时应支持 template 渲染，适配深色和浅色菜单栏。

## 4. 方案取舍

### 方案 A：全屏透明度淡入淡出

通过调整整个遮罩窗口透明度实现变黑/变亮。

- 优点：实现简单、风险低、性能稳定。
- 缺点：情绪表达弱，缺少“从四周扩散/收拢”的方向感。

### 方案 B：边缘向中心的渐变遮罩转场（推荐）

通过径向或多层渐变遮罩控制黑色覆盖区域，让边缘先变黑、中心后变黑；退出时边缘先变亮、中心后变亮。

- 优点：最贴近用户描述，有明显仪式感，仍然足够轻量。
- 缺点：需要增加转场状态和动画时序，关闭窗口不能再直接移除。

### 方案 C：复杂视觉特效转场

加入粒子、波纹、碎片化等更强视觉效果。

- 优点：冲击力最强。
- 缺点：实现复杂，可能影响多显示器同步和稳定性，也不符合工具型应用的克制定位。

V2 推荐采用方案 B。

## 5. 功能需求

### 5.1 黑屏开启动画

- 当用户通过菜单栏或快捷键开启黑屏时，目标屏幕不应硬切为纯黑。
- 黑色遮罩应从屏幕四周边缘开始出现，并在短时间内向屏幕中心推进，最终变为当前 V1 的稳定黑屏状态。
- 推荐动画时长为 220ms 到 420ms，体感应偏快速、有冲击力，但不能造成明显卡顿。
- 动画完成前，遮罩窗口必须已经拦截鼠标和键盘事件，避免过渡期间点击穿透到底层应用。
- 多显示器同时开启时，各屏幕动画应尽量同步，避免一个屏幕已经全黑而另一个屏幕仍未开始。

### 5.2 退出变亮动画

- 当用户退出黑屏时，不应直接移除遮罩窗口。
- 屏幕应从四周边缘开始恢复可见，并向中心推进，最终完全显示底层桌面或应用。
- 退出动画期间，遮罩仍应保持事件拦截，直到动画完成后再释放窗口。
- Esc、安全退出、菜单栏退出和快捷键退出都应走同一套退出转场，除非系统要求降级。

### 5.3 动效降级与安全兜底

- 如果系统开启“减少动态效果”，V2 应允许降级为更短、更平稳的淡入淡出，或直接使用无动画切换。
- 如果动画过程中出现屏幕参数变化、窗口重建或应用退出，遮罩状态必须保持可控，不允许残留无法关闭的黑色窗口。
- Esc 仍然必须是最高优先级的安全退出方式。
- V2 不新增动画开关设置，避免设置项膨胀；是否需要自定义动效留给后续版本评估。

### 5.4 中文界面范围

V2 需要覆盖以下运行时界面：

- 菜单栏按钮 tooltip。
- 菜单项：开启/退出黑屏、覆盖范围、快捷键状态、设置、退出应用。
- 设置窗口标题。
- 设置页标题、副标题、安全边界提示。
- 设置页分组标题、选择器、开关、说明文字、恢复默认按钮。
- 覆盖范围和内容模式枚举展示名称。
- 默认状态文字和空自定义文字兜底文案。
- App bundle 展示名和菜单栏主入口名。

文档、源码注释、构建脚本输出和包名不属于本次中文化范围，除非后续实现时发现它们会直接出现在用户运行界面中。

### 5.5 小黑马图标范围

- 项目应包含可复现的矢量源图。
- 打包产物应包含 `.icns` app 图标。
- 菜单栏应优先显示图标；图标加载失败时回退显示「黑码码」文字。
- 图标不依赖一次性 AI 生成图片作为唯一源文件。

## 7. 非功能需求

- 动画不得引入新的敏感系统权限。
- 动画不得改变 V1 的默认覆盖范围：默认仍为仅外接显示器。
- 稳定黑屏状态下仍应是纯黑背景，不能因为转场层残留导致边缘发灰或漏光。
- 动画过程应避免明显闪烁、白屏、窗口层级跳动。
- 中文文案应避免过长导致设置页按钮、菜单项或选择器截断。
- 现有设置数据应兼容，升级到 V2 后不应重置用户设置。
- 小黑马图标在 16px、32px 和 app icon 尺寸下都应保持可辨识。

## 8. 验收标准

1. 开启黑屏时，用户能看到黑色从屏幕边缘向中心推进，最终进入纯黑遮罩。
2. 退出黑屏时，用户能看到画面从屏幕边缘向中心恢复，动画结束后底层应用正常可见。
3. 动画期间鼠标和键盘事件不会穿透到被遮住的应用。
4. Esc 在稳定黑屏和动画过程中都能触发退出流程。
5. 外接显示器模式和所有显示器模式都支持同样的转场效果。
6. 设置页、菜单栏、遮罩默认状态文案都显示为中文，除快捷键名和必要内部兼容名外，不应出现面向用户的英文界面文案。
7. 菜单栏显示小黑马图标，加载失败时回退显示「黑码码」。
8. 打包产物包含小黑马 `.icns` 图标，Info.plist 展示名为「黑码码」。
9. `swift run VibeBlankCoreChecks` 和 `swift build --product VibeBlank` 仍能通过。

## 9. 非目标

V2 不包含以下内容：

- 系统锁屏、密码解锁、Touch ID 解锁。
- 真正关闭显示器电源或调节显示器亮度硬件。
- 自定义动画方向、动画速度或主题皮肤。
- 多语言运行时切换。
- 完整品牌视觉系统、营销官网或复杂插画资产。
- 触发角、定时黑屏、团队管理、账号体系或通知集成。

## 10. 实现提示

后续实现可优先考虑在 `OverlayContentView` 或独立转场视图中增加动画状态，由 `OverlayManager` 管理开启和退出阶段。退出时不应立即 `orderOut`，而应等待退出动画完成后再释放窗口。

如采用 SwiftUI 实现，可用径向渐变、遮罩层或可动画 shape 表达边缘向中心的推进；如 SwiftUI 表现受限，可在 `OverlayWindow` 的内容视图中引入轻量 Core Animation layer。无论采用哪种实现，事件拦截应继续由 `OverlayWindow` 保证。

小黑马图标建议以 SVG 作为源文件，再生成 `.icns` 供 macOS app bundle 使用。菜单栏图标应作为 template image 使用，以便系统自动适配菜单栏颜色。
